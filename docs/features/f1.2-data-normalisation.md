# F1.2 — Data Normalisation & Alignment

**Epic:** [E1 — Data Ingestion & API](../epics/e1-data-ingestion.md)

## Summary

Raw data from different providers arrives at different frequencies (daily, monthly, quarterly) and in different units. This feature normalises all series to a common monthly cadence and aligns date ranges so any two series can be divided point-by-point.

## Rules

- **Frequency**: all series resampled to monthly (end-of-month)
- **Downsampling**: daily → monthly using last value of each month
- **Upsampling**: quarterly → monthly using forward-fill
- **Alignment**: when computing a ratio, use the intersection of both series' date ranges
- **Gap handling**: missing interior values forward-filled (max 3 months); leading/trailing NaN dropped
- **Units**: raw values preserved — no index conversion at this layer (done in the ratio calculation)

## Long-run average calculation

For each ratio (numerator ÷ denominator), the long-run average is the arithmetic mean over the full aligned history. Exposed alongside the series in the API response.
