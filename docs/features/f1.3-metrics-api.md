# F1.3 — Metrics REST API

**Epic:** [E1 — Data Ingestion & API](../epics/e1-data-ingestion.md)

## Summary

Two endpoints expose metric data to the frontend. All business logic (fetching, normalisation, ratio calculation) lives in the service layer; routes are thin.

## Endpoints

### `GET /api/metrics`
Returns the full catalogue of available metrics.

**Response:**
```json
[
  { "id": "uk-house-prices", "label": "UK House Prices", "unit": "Index (£)", "isIndicatorOnly": false },
  { "id": "cape", "label": "Shiller CAPE Ratio", "unit": "Ratio", "isIndicatorOnly": true }
]
```

---

### `GET /api/metrics/{id}`
Returns the raw time series for a single metric.

**Query params:** `from` (ISO date, optional), `to` (ISO date, optional)

**Response:**
```json
{
  "id": "gold",
  "label": "Gold Price",
  "unit": "$/oz",
  "longRunAverage": 842.3,
  "series": [
    { "date": "1990-01-31", "value": 410.5 }
  ]
}
```

---

### `GET /api/metrics/ratio`
Computes and returns a ratio of two metrics.

**Query params:** `numerator`, `denominator`, `from` (optional), `to` (optional)

**Response:**
```json
{
  "numerator": "uk-house-prices",
  "denominator": "uk-wages",
  "label": "UK House Prices / UK Wages",
  "longRunAverage": 5.2,
  "series": [
    { "date": "1990-01-31", "value": 3.8 }
  ]
}
```
